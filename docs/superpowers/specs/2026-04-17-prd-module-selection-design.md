# PRD 内容模块选择功能设计

## 问题

当前 `prd-gen` skill 在生成 PRD 时会固定生成所有章节（第0章到第9章），但用户实际使用中并非所有章节都需要。特别是技术方案和接口定义等内容，用户明确表示通常不需要出现在 PRD 中。

## 目标

在 PRD 生成前增加用户确认环节，让用户选择需要生成的内容模块，并明确排除不适用的内容。

## 方案

### 方案选择

采用**方案 C：扩展 PRD Plan** — 在现有的 Step 0.5（信息缺口分析与 PRD Plan）环节中增加章节选择功能。这是最小改动方案，复用已有的用户审批流程。

### 具体改动

**文件：** `skills/prd-gen/SKILL.md`

#### 改动 1：在 Step 0.5 后新增章节选择步骤

在 "Information Gap Analysis & PRD Plan" 之后、用户审批之前，新增章节选择环节：

1. 使用 `AskUserQuestion`（multiSelect）展示所有章节供用户勾选
2. 默认勾选：第0-6章（行业对标、项目概述、业务分析、功能需求、非功能需求、用户体验流程、项目风险）
3. 默认不勾选：第7章（合规建议）、第8章（原型设计）、第9章（成功指标）
4. 同时在界面中明确告知用户：技术方案和接口定义不会包含在 PRD 中

#### 改动 2：修改 PRD Plan 展示格式

PRD Plan 表格中增加「选中状态」列，清晰展示哪些章节会被生成：

```markdown
### PRD 内容方案

| 章节 | 内容概要 | 选中 | 依赖状态 |
|:-----|:---------|:-----|:---------|
| 第0章 行业对标 | ... | ✅ | 已完成 |
| 第1章 项目概述 | ... | ✅ | 已完成 |
...
```

#### 改动 3：在生成指令中加入排除约束

在 PRD 生成 prompt 中追加：

```
## 排除内容
以下章节未被选中，请勿生成：[动态列出未选中的章节]

以下类型的内容不应出现在 PRD 中：
- 技术方案 / 架构设计
- API 接口定义 / 端点规格
- 数据库表结构设计
```

#### 改动 4：更新 Quality Gate Checklist

在质量检查清单中增加动态检查：只验证用户选中的章节是否完整。

### 涉及的文件

- `skills/prd-gen/SKILL.md` — 主要改动

### 对其他工作流的影响

`quick-prd`、`full-pm-cycle`、`feature-launch` 等工作流都调用 `prd-gen` skill，改动会自动生效，无需额外修改。
