# Data Monitor 知识库

## 指标基线

### 按行业/产品类型
- **SaaS B2B**:
  - DAU/MAU 比: 20-40% 为健康
  - 流失率月均: < 5%
  - NPS: > 40 为良好

- **消费应用**:
  - 次日留存: > 40%
  - 7日留存: > 20%
  - 30日留存: > 10%

## 常见异常模式

### 突增 (Spike)
- **原因**: 营销活动、病毒传播、数据问题
- **验证**: 检查流量来源、活动日历

### 突降 (Drop)
- **原因**: 部署问题、服务中断、季节性
- **验证**: 检查部署记录、服务状态

### 趋势变化
- **原因**: 产品方向变化、竞争影响、市场变化
- **验证**: 分析用户行为、竞品动态

## 有效告警阈值

### 设置原则
- 使用历史数据的 3σ 作为边界
- 考虑业务周期性 (周末/节假日)
- 分层告警: warning / critical

### 示例
- DAU: 低于历史均值 15% 触发警告
- 转化率: 低于历史均值 20% 触发严重告警
- 错误率: 高于 1% 触发告警

## 数据源可靠性

### 优先级
1. 产品埋点数据 (最准确)
2. 分析平台 (Google Analytics, Mixpanel)
3. 第三方追踪 (需验证)
