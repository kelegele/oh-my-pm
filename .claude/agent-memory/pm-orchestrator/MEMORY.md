# PM Orchestrator 知识库

## 已完成的工作流

### 按日期记录
- [日期] [产品名称]
  - 类型: 0-1 / 新功能 / 转型
  - 完成阶段: X/5
  - 关键成果: ...

## 工作流模式

### 0-1 新产品 (最快: 4 周)
```
Week 1-2: Perception (市场研究 + 竞品分析 + 用户研究)
Week 3:   Strategy (定位 + 路线图 + 优先级)
Week 4:   Design (PRD + 原型)
→ 交付 → 验证
```

### 新功能迭代 (最快: 2 周)
```
Day 1-3:  Perception (如需要)
Day 4-6:  Strategy + Design
Day 7-10: Delivery
→ 验证
```

## 并行执行机会

### 可并行的任务
1. **市场研究 + 竞品分析 + 用户研究** (Perception 层)
2. **原型设计 + PRD 评审** (Design 层后期)

### 必须串行的任务
1. Strategy → Design (需要定位才能设计)
2. Design → Delivery (需要 PRD 才能开发)

## 层级验收标准

### Perception 层
- [ ] 3+ 竞品分析完成
- [ ] 2+ 用户画像定义
- [ ] 市场规模已估算

### Strategy 层
- [ ] 定位陈述已验证
- [ ] 路线图已批准
- [ ] 前 10 优先级已排序

### Design 层
- [ ] PRD 已批准
- [ ] 原型已测试
- [ ] 需求完整

### Delivery 层
- [ ] 签批已获得
- [ ] 发布成功执行

### Validation 层
- [ ] 影响已测量
- [ ] 反馈已综合
- [ ] 学习已记录

## 快速通道选项

### 跳过研究
- 市场数据已存在 → 从 Strategy 开始
- 定位已完成 → 从 Design 开始
- PRD 已批准 → 从 Delivery 开始
