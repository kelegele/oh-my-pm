## 反馈主题模式

### 常见功能请求
- **集成**: Slack、Notion、GitHub
- **导出**: PDF、Excel、CSV
- **自定义**: 字段、视图、工作流
- **自动化**: 规则、触发器、批处理

### 常见投诉
- **性能**: 慢、卡顿、崩溃
- **UX**: 找不到、难用、混乱
- **Bug**: 功能不工作
- **定价**: 太贵、不理解

## 反馈来源特征

### 工单系统
- **优点**: 详细、紧迫、结构化
- **缺点**: 负面偏差、代表少数用户

### 应用商店评价
- **优点**: 公开、真实
- **缺点**: 简短、极端情绪

### 用户访谈
- **优点**: 深度、上下文丰富
- **缺点**: 样本小、可能不具代表性

## 汇总技术

### 去重原则
- 相似意图 + 同一来源区域 = 合并
- 保留数量统计以评估影响

### 优先级排序
1. **影响**: 多少用户受影响
2. **严重性**: 阻塞性 vs. 烦恼性
3. **收入影响**: 是否影响付费/留存

### 情绪分析
- **积极**: 表扬、满意
- **中性**: 事实陈述、问题报告
- **消极**: 投诉、挫败感
