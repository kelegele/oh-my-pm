# HTML 原型质量标准

## 什么是"优秀的原型图"

优秀的 HTML 原型图应该具备以下特质：

### 1. 视觉质量 (Visual Quality)

| 标准 | 说明 | 检查方法 |
|:-----|:-----|:---------|
| **设计一致性** | 颜色、字体、间距在整个原型中保持一致 | 对比多个页面的样式 |
| **层级清晰** | 通过大小、颜色、间距建立明确的视觉层级 | 一眼能看出什么是重要的 |
| **对齐精确** | 所有元素按网格对齐，无像素偏差 | 使用浏览器开发工具检查 |
| **留白适当** | 元素之间有适当的呼吸空间 | 内容不拥挤，布局舒适 |
| **配色和谐** | 主色、辅色、中性色搭配协调 | 使用配色工具验证 |

### 2. 交互质量 (Interaction Quality)

| 标准 | 说明 | 检查方法 |
|:-----|:-----|:---------|
| **反馈即时** | 点击、悬停有即时视觉反馈 | 交互后 100ms 内有响应 |
| **状态清晰** | 正常/悬停/激活/禁用状态明确 | 四种状态都能区分 |
| **过渡流畅** | 状态切换有平滑过渡动画 | 动画时长 150-300ms |
| **导航直观** | 用户知道自己在哪、能去哪 | 当前位置有明确指示 |
| **操作可撤销** | 重要操作提供取消/返回选项 | 有关闭/取消按钮 |

### 3. 代码质量 (Code Quality)

| 标准 | 说明 | 检查方法 |
|:-----|:-----|:---------|
| **语义化 HTML** | 使用正确的 HTML 标签 | header/nav/main/footer |
| **结构清晰** | 代码缩进正确，注释适当 | 人工审查 |
| **无语法错误** | HTML/CSS/JS 无错误 | 浏览器控制台无报错 |
| **性能优化** | 无不必要的资源加载 | 页面加载 < 2 秒 |
| **可访问性** | 支持键盘导航，ARIA 标签完整 | Tab 键可遍历所有交互元素 |

### 4. 保真度匹配 (Fidelity Match)

| 原型类型 | 用途 | 质量标准 |
|:---------|:-----|:---------|
| **Wireframe** | 结构验证、信息架构 | 灰度、简洁、无视觉干扰 |
| **Mockup** | 视觉设计评审、品牌对齐 | 完整配色、字体、间距 |
| **Interactive** | 用户测试、开发参考 | 完整交互、状态、反馈 |

## 质量检查流程

### 生成前检查

```markdown
- [ ] PRD 内容已完整阅读
- [ ] 用户流程已映射（页面列表 + 跳转关系）
- [ ] 设计风格已确认（颜色、字体、组件库）
- [ ] 保真度级别已选择（wireframe/mockup/interactive）
```

### 生成中检查

```markdown
- [ ] 使用正确的模板文件
- [ ] 组件来自标准组件库
- [ ] CSS 变量定义完整
- [ ] 响应式断点已添加
```

### 生成后检查

```markdown
- [ ] 在浏览器中打开无错误
- [ ] 所有链接/按钮可点击
- [ ] 页面切换流畅
- [ ] 移动端布局正常（F12 切换视图）
- [ ] 控制台无报错
```

## 质量评分表

使用以下评分表对生成的原型进行自评：

### 视觉质量 (0-10 分)

| 分数 | 标准 |
|:-----|:-----|
| 0-3 | 布局混乱，元素未对齐，颜色冲突 |
| 4-6 | 基本对齐，配色可用，无明显问题 |
| 7-8 | 设计一致，层级清晰，留白适当 |
| 9-10 | 专业级设计，可直接用于演示 |

### 交互质量 (0-10 分)

| 分数 | 标准 |
|:-----|:-----|
| 0-3 | 无交互，或交互有明显 bug |
| 4-6 | 基本交互可用，缺少状态反馈 |
| 7-8 | 交互流畅，状态完整，有过渡动画 |
| 9-10 | 交互体验优秀，接近真实产品 |

### 代码质量 (0-10 分)

| 分数 | 标准 |
|:-----|:-----|
| 0-3 | 代码混乱，有语法错误 |
| 4-6 | 代码可运行，结构一般 |
| 7-8 | 代码整洁，语义化好，有注释 |
| 9-10 | 代码质量高，可直接交付开发 |

### 综合评分

```
总分 = 视觉质量 × 0.4 + 交互质量 × 0.4 + 代码质量 × 0.2

9-10 分：优秀原型，可直接使用
7-8 分：  良好原型，建议优化
5-6 分：  可用原型，需要改进
0-4 分：  不可用，需要重做
```

## 常见问题与解决方案

### 问题 1：布局在移动端错乱

**原因**：未添加 viewport 或媒体查询

**解决**：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

```css
@media (max-width: 640px) {
    .container { padding: 16px; }
    .grid { grid-template-columns: 1fr; }
}
```

### 问题 2：按钮点击无反馈

**原因**：缺少 hover/active 状态

**解决**：
```css
.btn {
    transition: all 0.2s;
}
.btn:hover {
    opacity: 0.9;
    transform: translateY(-1px);
}
.btn:active {
    transform: translateY(0);
}
```

### 问题 3：颜色对比度不足

**原因**：前景色和背景色太接近

**解决**：使用对比度检查工具（如 WebAIM Contrast Checker），确保文本与背景对比度 ≥ 4.5:1

### 问题 4：键盘无法访问

**原因**：非原生元素缺少 tabindex

**解决**：
```html
<div role="button" tabindex="0" onclick="handleClick()">点击我</div>
```

## 原型交付清单

交付原型时，应包含以下内容：

```
context/prototypes/{feature-name}/
├── index.html           # 主原型文件
├── styles.css           # 自定义样式（如有）
├── script.js            # 自定义脚本（如有）
├── README.md            # 使用说明
└── screenshots/         # 预览截图
    ├── desktop.png
    ├── tablet.png
    └── mobile.png
```

### README.md 模板

```markdown
# {功能名称} 原型

## 预览方式

1. 双击 `index.html` 在浏览器中打开
2. 或运行：`open index.html`

## 功能说明

- 页面 A → 页面 B：点击 XX 按钮
- 弹窗演示：点击"详情"按钮
- 表单提交：填写后点击"提交"

## 注意事项

- 数据为模拟数据
- 部分交互为演示效果
- 移动端建议使用 Chrome 设备模拟

## 设计说明

- 主色调：#667eea
- 字体：系统默认字体
- 组件库：参考 COMPONENTS.md
```
